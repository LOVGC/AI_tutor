# understand

A personal learning skill: build a deep, **runnable mental model** of a topic — to understand it, not just memorize it. Invoke with `/understand <topic>`.

It is a fusion of two sources kept in the repo root:

- `../matt_teach_skill/` — a general, skills-oriented teaching skill (the structural machinery).
- `../my_own_learning/` — my own distilled learning philosophy (Feynman-style understanding).

## In one line

Matt's durable, beautiful, multi-session workspace — re-pointed from *drill a skill* to *build a runnable mental model*, with the Feynman cognitive loop as its core and the agent acting as examiner.

## The design decisions that shaped it

- **Aim:** understanding / mental models, not skill acquisition. Knowledge is the goal.
- **Method (the spine):** why-it-exists → map the *generative core* → build a runnable sensory model (first principles) → verify against sources → test in dialogue → space it.
- **Generative core:** the highest *useful* abstraction — the framework a field treats as bedrock, such that concrete techniques are implementations / special cases / patches (修修补补) of it. Done when the field stops feeling like a pile of separate facts.
- **Pillars:** Understanding · Durability (understanding-tests) · Expert landscape.
- **Retention engine:** understanding-tests run *in dialogue* (predict-then-check, Feynman explain-back, spaced re-explanation) — not skill drills, and not quizzes inside lessons.
- **Lessons:** stay polished, beautiful, self-contained, richly visual; each installs one piece of the model. Testing happens in dialogue, not inside them.
- **Grounding:** first principles to build the model, trusted sources to verify it.
- **Dropped from Matt:** skill drills, equal-length quiz answers, skill-interleaving, and explicit emotional-stance coaching.
