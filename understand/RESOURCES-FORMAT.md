# RESOURCES.md Format

`RESOURCES.md` is the curated set of trusted sources for the topic, plus the map of the field's expert landscape. The model is *built* from first principles but *verified* here — never trust parametric memory for a factual claim.

## Structure

```md
# {Topic} Resources

## Foundational sources
{High-trust material to verify and deepen the model against. Feed these to yourself before reasoning about facts. Prefer primary sources, recognised experts, textbooks, papers, lecture notes.}

- [Book: {title} — {author}]({url})
  What it covers; when to reach for it.
- [Paper / lecture / article: {title} — {author}]({url})
  What it covers; when to reach for it.

## Expert landscape
{The shape of the field's knowledge. Fill this in as it emerges — it is how you find and stress-test the generative core.}

- **Generative core / shared mental models:** the foundational framework(s) every expert treats as bedrock. (Cross-link to MENTAL-MODEL-MAP.md.)
- **Fault lines:** where experts genuinely disagree, with the best argument from each side.
- **Open problems:** what is still unsolved or actively contested.

## Communities (optional)
- [{community}]({url}) — when to reach for it.

## Gaps
- {Areas the mission needs where no good source has yet been found — this drives future search.}
```

## Rules

- **High-trust only.** Primary sources, recognised experts, peer-reviewed work. If a resource is marketing dressed as education, leave it out.
- **Annotate every entry.** A bare link is useless in three months. One line: what it covers, when to reach for it.
- **Map disagreement, not just consensus.** The fault lines are where the field is alive. Recording the best argument on each side is itself deep understanding.
- **The landscape prompts.** To populate that section, ask of the sources: *What are the fundamental mental models every expert shares? Where do they fundamentally disagree? What's the best argument on each side? What are the open problems?*
- **Surface gaps explicitly.** If no good source exists for an area the mission needs, list it under Gaps.
- **Prune ruthlessly.** Remove sources that turn out wrong, shallow, or off-mission. Better five sharp sources than thirty mediocre ones.
- **Record community preferences.** If the user prefers not to join communities, note it so future sessions don't keep proposing them.
